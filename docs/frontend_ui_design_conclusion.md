# OneBook AI 前端界面设计结论（贴合题目）

## 1. 输入问题（原始题目）
本项目是一个面向个人书库的 AI 对话问答系统，核心能力是：
- 上传电子书（PDF/EPUB/TXT）
- 解析书籍内容并可基于内容提问
- 具备书库管理能力
- 具备会话历史记录能力

## 2. 设计方法（基于 ui-ux-pro-max）
本次采用 `ui-ux-pro-max` 进行检索与设计系统归纳，关键结论来自以下方向：
- `--design-system`：给出整体模式、颜色与字体建议
- `--domain style`：偏向 `AI-Native UI` 与 `Swiss Modernism 2.0`
- `--domain color`：`Knowledge Base/Documentation` 配色最贴近本项目
- `--domain typography`：`Inter` 与 `IBM Plex Sans` 都适合文档/工具型产品

## 3. 设计定位
- 产品类型定位：`Knowledge Base + AI Assistant`
- 视觉定位：`Swiss Modernism 2.0`（清晰网格、轻装饰）+ `AI-Native UI`（对话优先）
- 主题策略：`浅色优先`（更利于答辩演示与文档阅读）
- 风格目标：专业、可信、高可读，不追求炫技动效

## 4. 信息架构（IA）
- `登录`：进入系统与用户身份确认
- `书库`：上传、解析状态、书籍管理入口
- `对话`：选择 ready 书籍后提问，显示引用与答案
- `会话历史`：按书籍和时间回看会话

## 5. 页面骨架规范
### 5.1 顶部全局区
- 固定顶栏：品牌、系统状态、主导航
- 导航保持 3 项：`书库` / `对话` / `会话历史`

### 5.2 书库页
- 顶部动作区：上传按钮 + 支持格式提示（PDF/EPUB/TXT）
- 状态统计区：`queued/processing/ready/failed`
- 书籍表格区：标题、格式、状态、更新时间、操作
- 处理流程区：上传 -> 解析 -> 可问答

### 5.3 对话页
- 左侧：可问答书籍列表 + 最近会话列表
- 右侧：消息区、引用来源区、输入区
- 强约束提示：仅 `ready` 书籍可问答

### 5.4 会话历史页
- 提供筛选条件（书籍、时间、关键字）
- 按会话条目展示：书名、时间、消息数、摘要、跳转入口

## 6. 视觉与可用性约束
- 字体：`IBM Plex Sans`（正文）+ `IBM Plex Mono`（状态码/技术信息）
- 颜色（核心 token）：
  - 背景：`#F8FAFC`
  - 文字：`#1E293B`
  - 主强调：`#2563EB`
  - 中性色：`#475569 / #64748B`
- 动效：仅保留轻量进入动效与 hover 过渡（150-250ms）
- 可访问性：
  - 表单必须有明确标签
  - 焦点态可见
  - 对动态更新区域使用 `aria-live`
